# Чекліст оператора: підключення нового викладача (тенанта)

Мета — за ≤30 хвилин підняти брендований додаток для викладача. Викладач надсилає
лише **назву**, **лого** і **перший список слів**. Решту робиш ти (оператор).

> Транспорт — polling. Після кроку 3 обовʼязково **Redeploy** на Railway, щоб бот
> запрацював. Ліміт BotFather — 20 ботів на акаунт; при масштабуванні — другий акаунт.

---

## Крок 1. Створити бота в BotFather

1. У Telegram відкрий [@BotFather](https://t.me/BotFather) → `/newbot`.
2. **Назва** бота = бренд викладача (напр. «Слова з Оксаною»).
3. **Username** — латиницею, закінчується на `bot` (напр. `slova_oksana_bot`).
4. Скопіюй **токен** (формат `1234567:AA...`) — він знадобиться на кроці 3.
5. Постав аватар: `/setuserpic` → надішли лого викладача.
6. (Опис/short-опис можна не чіпати — бот сам їх не встановлює.)

## Крок 2. Кнопка-меню з Mini App

У BotFather: `/mybots` → обери бота → **Bot Settings → Menu Button → Configure menu
button** → введи URL:

```
https://miniapp-omega-three.vercel.app
```

(Той самий URL для всіх — бренд визначається автоматично з підпису бота.)

## Крок 3. Завести тенанта скриптом

На своїй машині (репозиторій `wordsnap`, тека `backend/`) або в Railway shell:

```bash
python -m scripts.add_tenant \
  --slug oksana \
  --display-name "Слова з Оксаною" \
  --bot-token 1234567:AA...   # токен з кроку 1 \
  --owner-telegram-id 469478065   # telegram_id викладача \
  --logo-url https://.../logo.png   # опційно \
  --color-primary '#0EA5E9' --color-accent '#22C55E'   # опційно
```

Скрипт перевірить токен (getMe), створить тенанта і надрукує підсумок + нагадування.
`--owner-telegram-id` — це числовий ID викладача в Telegram (можна дізнатись через
[@userinfobot](https://t.me/userinfobot)).

## Крок 4. Redeploy

Railway → сервіс бота → **Redeploy**. Після старту в логах має бути рядок
`🤖 Tenant bot up: oksana (id=…)`.

## Крок 5. Призначити викладачу роль teacher

Викладач має **один раз натиснути Start** у своєму боті (щоб зʼявитись у БД).
Потім признач роль (Railway shell / psql):

```sql
UPDATE users SET role='teacher'
 WHERE telegram_id = 469478065      -- owner_telegram_id викладача
   AND tenant_id = (SELECT id FROM tenants WHERE slug='oksana');
```

Після цього у викладача в Mini App зʼявиться вкладка **«Викладач»**.

## Крок 6. Смоук-тест

- [ ] Відкрий бота викладача → Start → бачиш вітання від бренду (не «WordSnap»).
- [ ] Відкрий Mini App → колір/назва/лого — бренду викладача.
- [ ] Немає жодних цін / екранів підписки.
- [ ] Викладач бачить вкладку «Викладач», може створити колоду.
- [ ] (Опційно) з тестового акаунта-«учня» відкрий того ж бота → бачиш призначені колоди.

## Готово

Передай викладачу памʼятку `teacher_welcome_ua.md` і посилання на його бота
(`https://t.me/slova_oksana_bot`).

---

### Часті питання

- **Другий викладач із тим самим Telegram-акаунтом-учнем?** OK — учень існує окремо
  в кожному тенанті (ізоляція за `(telegram_id, tenant_id)`).
- **Передати бота викладачу пізніше?** BotFather → `/mybots` → Transfer Ownership.
- **Поставити на паузу?** `UPDATE tenants SET plan='paused' WHERE slug='oksana';`
  + Redeploy (бот перестане підніматись).
